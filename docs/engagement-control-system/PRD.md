# AI 短剧生成系统 v2.0 - 产品需求文档

**版本：v2.0 | 状态：待评审**
**创建日期：2026-03-01**

---

## 1. 背景与目标

### 1.1 当前系统现状

inkplotWorkshop 是一个 AI 驱动的短剧生成系统,当前已实现:
- ✅ 基础工作流:输入 → 故事设计 → 资产管理 → 剧本设计 → 分镜设计 → 导出
- ✅ 项目管理、剧本编辑、资产库、分镜生成
- ✅ 敏感词审查、美术风格配置

### 1.2 核心问题

当前系统在**叙事连贯性**和**用户留存**方面存在明显不足:

| 问题类型 | 具体表现 |
|---------|---------|
| **剧集割裂** | 集与集之间缺乏状态延续,角色动机突变 |
| **爽点失控** | 爽点释放无规律,导致情绪脱敏或压抑过载 |
| **钩子缺失** | 集尾缺乏有效悬念,追更率不稳定 |
| **资产浪费** | 场景/角色过多,视觉一致性差 |

### 1.3 v2.0 核心目标

**通过引入状态机制、内流控制层和资产约束,实现:**

1. **叙事连贯性** - 角色动机、事件状态跨集延续
2. **情绪节奏控制** - 系统级爽点调度与钩子契约
3. **视觉一致性** - 资产复用率提升,每集资产≤3
4. **电影质感** - 自动适配电影滤镜与导演思维提示词

## 3. 功能需求详述

### 3.1 故事设计层升级

#### 3.1.1 动态利益矩阵 (Interest Matrix)

**需求描述:**
在生成"不分集的故事"时,为每个角色建立结构化的利益矩阵。

**数据结构:**
```typescript
interface CharacterMatrix {
  id: string;
  name: string;
  identity: string;      // 身份 (e.g. "隐藏富豪")
  secret: string;        // 秘密 (e.g. "真实身份是集团继承人")
  motivation: string;    // 核心欲望 (e.g. "证明自己不靠家族")
  asset: string[];       // 筹码 (e.g. ["财富", "人脉", "商业头脑"])
  motivationLocked: boolean; // 动机锁定状态
  motivationUnlockEvent?: string; // 解锁事件ID
}
```

**约束规则:**
- **连贯性约束:** 在10集内,除非触发特定"事件状态:升级",否则角色核心欲望禁止突变
- **利益交汇点锁定:** 系统预先生成 `Conflict_Graph`,明确 A 与 B 在第几集必须产生利益摩擦

**UI 需求:**
- 在项目创建/编辑时,展示角色利益矩阵编辑器
- 可视化冲突拓扑图 (节点=角色,边=冲突关系+触发集数)

---

#### 3.1.2 冲突拓扑图 (Conflict Graph)

**需求描述:**
预先规划角色间的冲突节点,作为后续生成的硬约束。

**数据结构:**
```typescript
interface ConflictNode {
  id: string;
  characterA: string;    // 角色A ID
  characterB: string;    // 角色B ID
  conflictType: 'interest' | 'secret' | 'revenge' | 'misunderstanding';
  triggerEpisode: number; // 必须在第几集触发
  status: 'pending' | 'triggered' | 'resolved';
}
```

### 3.2 资产设计层升级 ★NEW

#### 3.2.1 资产前置约束

**需求描述:**
资产设定提前到剧本生成前,限制每集资产总数≤3。

**约束规则:**
- 每集最多使用 3 个资产 (人物+场景+道具总和)
- 优先复用已有资产,减少新资产创建
- 尽量在单一场景内完成一集内容

**数据结构扩展:**
```typescript
interface Episode {
  // ... 现有字段
  assetBudget: {
    maxAssets: number;        // 默认 3
    usedAssetIds: string[];   // 本集使用的资产ID
    primaryLocation: string;  // 主场景ID
  };
}
```

---

#### 3.2.2 电影滤镜自动适配

**需求描述:**
根据题材和故事,自动适配电影滤镜风格,提升视觉质感。

**滤镜类型:**
| 题材 | 推荐滤镜 | 色调特征 |
|------|---------|---------|
| 逆袭爽剧 | 都市冷峻 | 高对比度,冷色调,锐利边缘 |
| 甜宠爽剧 | 柔光暖调 | 柔焦,暖色调,高饱和度 |
| 悬疑短剧 | 黑色电影 | 低饱和度,强阴影,蓝绿色调 |
| 古装仙侠 | 水墨写意 | 低饱和度,雾气感,中国传统色 |

**数据结构扩展:**
```typescript
interface Project {
  // ... 现有字段
  cinematicFilter?: {
    name: string;           // 滤镜名称
    colorGrading: string;   // 调色描述
    lightingStyle: string;  // 光照风格
    moodKeywords: string[]; // 情绪关键词
  };
}
```

### 3.3 剧本设计层升级

#### 3.3.1 事件状态缓冲器 (Event State Buffer)

**需求描述:**
强制 AI 在生成下一集概述前,读取上一集的事件状态,确保连贯性。

**数据结构:**
```typescript
interface EventState {
  id: string;
  type: 'mainline' | 'subplot';
  description: string;
  status: 'pending' | 'active' | 'resolved';
  introducedInEpisode: number;
  resolvedInEpisode?: number;
  relatedCharacters: string[]; // 角色ID列表
}

interface Episode {
  // ... 现有字段
  eventStates: {
    inherited: EventState[];  // 从上集继承的事件
    introduced: EventState[]; // 本集新引入的事件
    resolved: EventState[];   // 本集解决的事件
  };
}
```

**状态转换规则:**
```
pending → active → resolved
   ↓         ↓
 (可跨集)  (必须在承诺期限内解决)
```

---

#### 3.3.2 内流控制层 (Engagement Control Layer)

**需求描述:**
集成已有的内流控制 PRD (见 `docs/n/内流控制_PRD_v2.md`),实现爽点调度、压抑指数、钩子契约。

**核心模块:**
1. **Payoff Ledger (爽点账本)** - 追踪爽点等级、释放状态
2. **Suppression Index (压抑指数)** - 量化观众"被憋程度"
3. **Hook Contract (钩子契约)** - 强制每集结尾悬念
4. **Release Scheduler (释放调度器)** - 统筹调度

**数据结构:** (详见内流控制 PRD)

**UI 需求:**
- 在剧本编辑器中显示当前集的:
  - SI 值 (压抑指数)
  - 计划释放的爽点等级
  - 钩子类型与强度
- 提供"节奏预览"图表,展示全剧 SI 走势

### 3.4 分镜设计层升级

#### 3.4.1 导演思维提示词结构 ★NEW

**需求描述:**
舍弃之前的 P0-P2 结构,采用导演思维的提示词结构。

**新提示词结构:**
```typescript
interface ShotPrompt {
  // 导演思维元素
  narrativeIntent: string;    // 叙事意图 (这个镜头要传达什么)
  emotionalTone: string;      // 情绪基调 (紧张/温馨/压抑)
  visualMetaphor?: string;    // 视觉隐喻 (可选)

  // 技术参数
  shotSize: string;           // 景别 (Close-up, Wide, etc.)
  cameraMovement: string;     // 运镜 (Pan, Tilt, Dolly, etc.)
  lighting: string;           // 光照 (自然光/戏剧性光/背光)

  // 电影滤镜 (继承自 Project)
  cinematicFilter: string;    // 滤镜描述

  // 画面描述
  visualDescription: string;  // 最终画面描述
  dialogue?: string;          // 对白
}
```

**示例:**
```json
{
  "narrativeIntent": "展现主角内心的孤独与决心",
  "emotionalTone": "压抑而坚定",
  "visualMetaphor": "窗外雨水模糊视线,象征前路不明",
  "shotSize": "Medium Close-up",
  "cameraMovement": "Slow push-in",
  "lighting": "侧光,半边脸在阴影中",
  "cinematicFilter": "都市冷峻 - 高对比度冷色调",
  "visualDescription": "男主站在落地窗前,背对镜头,窗外暴雨...",
  "dialogue": "\"这次,我不会再输了。\""
}
```

#### 3.4.2 自动过渡镜头 (Extension Shots)

**需求描述:**
在关键转场处自动插入过渡镜头,提升转场自然度。

**触发条件:**
- 场景切换 (Location A → Location B)
- 时间跳跃 (白天 → 夜晚)
- 情绪转折 (紧张 → 平静)

**过渡镜头类型:**
| 类型 | 描述 | 示例 |
|------|------|------|
| Establishing Shot | 建立镜头 | 城市全景 → 具体建筑 |
| Match Cut | 匹配剪辑 | 关门动作 → 开门动作 |
| J-Cut / L-Cut | 声音先行/滞后 | 下一场景声音提前出现 |
| Transition Object | 过渡物体 | 特写某物体,淡入下一场景 |

**数据结构:**
```typescript
interface TransitionShot extends Shot {
  transitionType: 'establishing' | 'match_cut' | 'j_cut' | 'l_cut' | 'object';
  fromShotId: string;  // 前一镜头ID
  toShotId: string;    // 后一镜头ID
  autoGenerated: true; // 标记为自动生成
}
```

## 4. 用户交互流程

### 4.1 项目创建流程 (更新)

```
1. 用户输入
   ├─ Chat 窗口自由输入
   └─ 或选择热门选题模板

2. 故事设计 ★NEW
   ├─ AI 生成初步故事大纲
   ├─ 自动生成角色利益矩阵
   ├─ 用户编辑/确认矩阵
   └─ 系统生成冲突拓扑图

3. 资产设计 ★NEW (前置)
   ├─ AI 提取核心资产 (≤10个)
   ├─ 自动适配电影滤镜
   └─ 用户确认资产库

4. 剧本设计
   ├─ 分集梗概生成 (带事件状态)
   ├─ 内流控制层介入 (爽点/钩子调度)
   ├─ 详细剧本生成 (每集≤3资产)
   └─ 敏感词审查

5. 分镜设计
   ├─ 导演思维提示词生成
   ├─ 自动插入过渡镜头
   └─ 用户微调

6. 导出
   └─ 生成最终视频/图片序列
```

---

### 4.2 剧本编辑器增强

**新增功能:**
1. **事件状态面板** - 显示当前集的事件状态 (pending/active/resolved)
2. **内流控制仪表盘** - 实时显示 SI 值、爽点计划、钩子状态
3. **资产预算提示** - 当前集已用资产数 / 3
4. **角色动机锁定提示** - 显示角色动机是否可变

## 5. 技术约束

### 5.1 性能要求
- 事件状态缓冲器读取延迟 < 100ms
- 内流控制层调度计算 < 500ms
- 过渡镜头生成不影响主流程 (异步)

### 5.2 数据一致性
- 角色利益矩阵变更需触发冲突图重算
- 事件状态变更需同步到所有相关集
- 资产删除需检查引用关系

| 指标 | 基准线 | 目标值 |
|------|--------|--------|
| 剧集连贯性评分 (人工评审) | - | ≥ 8/10 |
| 资产复用率 | - | ≥ 60% |

| 钩子兑现率 | - | ≥ 95% |
| 用户满意度 (生成质量) | - | ≥ 4.5/5 |
