# 甜宠爽剧 · 内流控制默认模板（10 集）

## 模板定位
- 题材：甜宠 / 契约婚姻 / 先婚后爱
- 核心目标：稳定观看时长 + 情感黏性追更
- 情绪机制：误会 → 靠近 → 甜 → 冲突 → 更甜

---

## 1. Payoff Ledger（情感爽点账本）

### 爽点预算
```json
{
  "S": 1,
  "A": 4,
  "B": 6
}
```

### 爽点语义约束
- S：公开官宣 / 生死级情感确认
- A：亲密升级 / 明确偏爱
- B：暧昧 / 日常甜点

---

## 2. Emotional Tension Index（情感张力）

### 权重配置
```json
{
  "misunderstanding": 0.3,
  "emotional_distance": 0.25,
  "external_obstacle": 0.25,
  "affection_withheld": 0.2
}
```

### 阈值
- A / S 前：最近 2 集平均张力 ≥ 0.55
- 连续 2 集无甜点 → 判定失败

---

## 3. Hook Contract（甜宠钩子）

### 允许类型
- emotional_delay（最高）
- misunderstanding
- affection_confirmation

### 示例
```json
{
  "hook_type": "emotional_delay",
  "withheld_affection": "告白",
  "payoff_level": "A",
  "scheduled_release": 5
}
```

- 钩子必须围绕关系变化
