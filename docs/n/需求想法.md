从【不分集的故事】到【分集的剧本】，这一步完成的是语义建构。大逻辑还遵从【总的故事—分集的故事梗概—单集展开梗概完成剧本撰写】。
- 加上状态机，量化地写在分集前的故事设计（人物）和每一集的梗概（人物+事件）里。【分集梗概+状态】
  - 【不分集的故事】：人物：
    - 第一是，完成动态利益矩阵设计，每个角色拥有 Identity（身份）、Secret（秘密）、Motivation（核心欲望）和 Asset（筹码）。
    - 第二是，要加上连贯性约束：在10集内，除非触发特定的“事件状态：升级”，否则角色的核心欲望禁止在中间剧集发生突变。
    - 第三是利益交汇点锁定：系统必须预先生成 Conflict_Graph（冲突拓扑图），明确 A 与 B 在第几集必须产生利益摩擦，作为后续生成的“硬约束”。
