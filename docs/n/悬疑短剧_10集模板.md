# 悬疑短剧 · 内流控制默认模板（10 集）

## 模板定位
- 题材：悬疑 / 推理 / 犯罪 / 阴谋
- 核心目标：提高完播率 + 强烈追更动机
- 情绪机制：信息差 → 怀疑 → 反转 → 更大谜团 → 真相揭示

> 悬疑剧不靠“爽点”，靠**信息控制**。  
> 系统目标不是让观众爽，而是让观众**不确定、焦虑、必须知道真相**。

---

## 1. Mystery Ledger（谜题账本）

### 1.1 谜题预算（10 集固定）

```json
{
  "core_mystery": 1,
  "secondary_mystery": 2,
  "red_herring": 4,
  "major_reversal": 2,
  "final_truth": 1
}
```

### 1.2 谜题语义定义

| 类型 | 定义 |
|----|----|
| core_mystery | 整季唯一核心真相（谁是凶手 / 真相是什么） |
| secondary_mystery | 阶段性谜题（某人动机 / 某线索真伪） |
| red_herring | 误导观众的假线索 |
| major_reversal | 推理方向被彻底推翻 |
| final_truth | 核心谜题完整揭示 |

⚠️ 原则：  
**10 集内只允许 1 个终极真相，其余都是通向它的错路。**

---

## 2. Information Asymmetry Index（信息差指数）

### 2.1 信息差权重配置

```json
{
  "audience_knows_more_than_protagonist": 0.25,
  "protagonist_knows_more_than_audience": 0.2,
  "hidden_evidence": 0.25,
  "false_assumption": 0.2,
  "timeline_gap": 0.1
}
```

> 信息差指数 ∈ [0, 1]

---

### 2.2 系统阈值规则

- **重大反转 / 真相释放前**：
  - 最近 2 集平均信息差 ≥ 0.6
- 若出现：
  - 信息差 < 0.4  
  - 且无新谜题生成  
→ 判定为【悬疑塌陷集】，禁止进入分镜

---

## 3. Hook Contract（悬疑钩子契约）

### 3.1 允许的钩子类型（仅限以下）

| 优先级 | 类型 | 说明 |
|----|----|----|
| P0 | revelation_interrupt | 关键发现被打断 |
| P1 | suspicion_shift | 怀疑对象突然转移 |
| P2 | evidence_contradiction | 新证据否定旧结论 |
| 禁用 | pure_danger | 只有危险无信息 |

⚠️ 悬疑剧禁止“无信息增量的危险结尾”。

---

### 3.2 钩子结构示例

```json
{
  "hook_type": "revelation_interrupt",
  "withheld_information": "真正的作案时间",
  "related_mystery": "core_mystery",
  "scheduled_release": 6
}
```

---

## 4. Revelation Scheduler（真相释放调度）

### 4.1 10 集推荐信息节奏

| 集数 | 信息策略 |
|----|----|
| 1 | 抛出核心谜题 |
| 2 | 提供关键线索 + 假设 |
| 3 | Red Herring |
| 4 | Secondary Mystery 解答 |
| 5 | 重大反转（方向错误） |
| 6 | 新核心假设 |
| 7 | Red Herring |
| 8 | Major Reversal |
| 9 | 真相前夜（只差一块） |
|10 | Final Truth |

---

### 4.2 系统硬限制

- 第 7 集前禁止揭示 Final Truth
- 每集至少新增或否定一个推理假设
- 连续 2 集未改变认知状态 → 强制重写

---

## 5. 失败模式自动拦截

系统直接判失败的情况：
- 真相在中段被完整揭示
- 多个“终极真相”并存
- 钩子只有追逐 / 打斗 / 危险
- 线索无法在后续回收

---

## 6. 成功判定信号（内流）

- 中段（4–7 集）点击率稳定不掉
- 第 8 集出现明显点击抬升
- 第 10 集完播率显著高于均值

---

## 7. 模板一句话定义

> **悬疑短剧不是告诉观众答案，  
而是不断告诉观众：你刚刚想错了。**
